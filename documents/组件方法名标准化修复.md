# ModernLineEdit 方法名修复

## 问题描述

运行 `all_components_demo.py` 时报错：
```
AttributeError: 'ModernLineEdit' object has no attribute 'set_placeholder_text'.
Did you mean: 'setPlaceholderText'?
```

## 原因

Demo代码使用了snake_case命名规范（`set_placeholder_text`），但ModernLineEdit只提供了Qt的camelCase方法（`setPlaceholderText`）。

## 解决方案

为ModernLineEdit添加snake_case便捷方法别名，与其他组件保持一致的命名风格。

### 添加的方法

| 新方法 (snake_case) | Qt原方法 (camelCase) | 功能 |
|-------------------|---------------------|------|
| `set_placeholder_text(text)` | `setPlaceholderText(text)` | 设置占位符文本 |
| `get_placeholder_text()` | `placeholderText()` | 获取占位符文本 |
| `set_echo_mode(mode)` | `setEchoMode(mode)` | 设置回显模式 |
| `get_echo_mode()` | `echoMode()` | 获取回显模式 |
| `set_read_only(read_only)` | `setReadOnly(read_only)` | 设置只读状态 |
| `is_read_only()` | `isReadOnly()` | 检查是否只读 |

### 使用示例

```python
from components.inputs.modern_line_edit import ModernLineEdit

lineedit = ModernLineEdit()

# 使用snake_case方法（推荐，一致性好）
lineedit.set_placeholder_text("Enter your name...")
lineedit.set_echo_mode(ModernLineEdit.EchoMode.Password)
lineedit.set_read_only(True)
if lineedit.is_read_only():
    print("Read-only mode")

# 或者使用Qt原方法（仍然可用）
lineedit.setPlaceholderText("Enter your name...")
lineedit.setEchoMode(ModernLineEdit.EchoMode.Password)
lineedit.setReadOnly(True)
```

## 命名规范

所有组件现在都支持两种命名方式：

1. **snake_case方法**（推荐）- 与项目代码风格一致
2. **camelCase方法** - Qt原生方法，向后兼容

这提供了更好的API一致性和开发者体验。
