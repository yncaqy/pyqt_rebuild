# 无边框窗口边缘调整完整修复

## 修复日期
2026-02-15

## 问题描述

### 核心问题
边缘调整大小的逻辑存在问题：
1. **达到最小尺寸后继续拖动会触发窗口移动**
2. **调整大小和窗口移动的逻辑混淆**

### 用户反馈
> "应该是拖动改变窗口大小的逻辑有问题，应该检查当前窗口如果到了最小，就不能再继续增减尺寸，也不能移动窗口，移动也应该是按住标题栏的时候"

### 根本原因

1. **调整大小逻辑不完善**
   - 达到最小尺寸后没有提前返回
   - 继续计算和应用几何变化
   - 可能导致状态混乱

2. **窗口移动触发条件不明确**
   - 只应在按住标题栏非边缘区域时触发
   - 调整大小时必须完全阻止窗口移动

## 修复方案

### 修复 1：改进调整大小逻辑

**文件：** `src/core/frameless_window.py` (第 526-588 行)

**关键改进：**

#### 1.1 提前检查是否在最小尺寸

```python
def _resize_window(self, global_pos: QPoint):
    """Resize window based on mouse movement."""
    try:
        if self._geometry is None:
            return

        delta = global_pos - self._press_pos
        geo = self._geometry

        # Minimum size constraints
        min_w = 400
        min_h = 300

        # Get current window geometry
        current_geo = self.geometry()
        current_w = current_geo.width()
        current_h = current_geo.height()

        # Check if already at minimum size and trying to shrink further
        if 'top' in self._edge or 'bottom' in self._edge:
            if current_h <= min_h:
                # Already at minimum height, check if trying to shrink
                if 'top' in self._edge and delta.y() > 0:
                    return  # ← 提前返回：从顶部缩小，停止
                if 'bottom' in self._edge and delta.y() < 0:
                    return  # ← 提前返回：从底部缩小，停止

        if 'left' in self._edge or 'right' in self._edge:
            if current_w <= min_w:
                # Already at minimum width, check if trying to shrink
                if 'left' in self._edge and delta.x() > 0:
                    return  # ← 提前返回：从左侧缩小，停止
                if 'right' in self._edge and delta.x() < 0:
                    return  # ← 提前返回：从右侧缩小，停止
```

**优点：**
- ✅ 达到最小尺寸后立即停止
- ✅ 避免不必要的计算
- ✅ 防止状态混乱

#### 1.2 只在几何变化时更新

```python
        # Calculate new geometry...
        # (same as before)

        # Only apply if geometry would actually change
```

**优点：**

## 技术要点

### 1. 提前返回模式

```python
# 好的做法：提前检查并返回
if current_h <= min_h and trying_to_shrink:
    return  # 不执行后续逻辑

# 避免：让逻辑继续执行
if current_h <= min_h and trying_to_shrink:
    new_top = ...  # 仍然计算
    # 最后再限制
```

**优点：**
- 代码更清晰
- 性能更好
- 避免副作用

### 2. 状态分离原则

```python
# FramelessWindow: 调整大小状态
self._resizing = True/False

# TitleBar: 拖动状态
self._dragging = True/False

# 互斥保护
if self._window._resizing:
    return  # 不执行拖动逻辑
```

### 3. 事件优先级

```
边缘调整大小 > 标题栏拖动 > 其他事件
```

实现方式：
- TitleBar 检测边缘位置并忽略事件
- 事件传播到 FramelessWindow 处理调整大小
- 调整大小时完全阻止拖动逻辑

## 设计原则总结

### 1. 明确的职责分离

- **FramelessWindow**：只负责边缘调整大小
- **TitleBar**：只负责窗口拖动
- 两者互斥，通过状态检查保护

### 2. 提前检查，提前返回

- 在执行操作前检查条件
- 不满足条件时立即返回
- 避免副作用和性能浪费

### 3. 多层保护机制

- **第一层**：事件入口检查（mousePressEvent）
- **第二层**：事件处理检查（mouseMoveEvent）
- **第三层**：状态重置检查（mouseReleaseEvent）

### 4. 用户体验优先

- 达到最小尺寸后立即停止响应
- 避免意外的窗口移动
- 清晰的光标反馈
