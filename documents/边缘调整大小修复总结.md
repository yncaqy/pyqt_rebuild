# FramelessWindow 边缘调整大小修复总结

## 修复日期
2026-02-14

## 问题描述

### 问题 1：标题栏无法拖动
**症状：** 按住标题栏无法移动窗口
**原因：** 重构时遗漏了 `setWindow()` 调用

### 问题 2：所有边缘无法调整大小
**症状：** 拖动窗口边缘无效
**原因：** 重构时删除了 `_resizing` 变量

### 问题 3：顶部边缘触发拖动而不是调整大小
**症状：** 鼠标在顶部边缘时触发的是窗口拖动而非调整大小
**原因：** TitleBar 优先接收事件，总是启动拖动逻辑

---

## 根本原因分析

### 事件传播机制

```
用户操作 → TitleBar (子组件) → FramelessWindow (父窗口)
```

在 Qt 中：
1. 子组件优先接收事件
2. 如果子组件处理事件（accept），事件不传播到父组件
3. 如果子组件忽略事件（ignore），事件传播到父组件

### 问题组件

重构时错误地用 `StateManager` 统一管理状态，导致：
1. **拖动状态**：`title_bar._state_mgr.pressed` - 标题栏拖动
2. **调整状态**：`window._state_mgr.pressed` - 被错误地用于边缘调整

这两个状态应该**独立**管理，不能共享。

#### 修复 7：添加顶部边缘检测方法
**位置：** `frameless_window.py:260-272`

```python
def _is_on_top_edge(self, pos: QPoint) -> bool:
    """检测是否在顶部边缘（用于调整大小）"""
    margin = self._edge_margin
    return pos.y() <= margin
```

#### 修复 8：TitleBar.mousePressEvent 添加优先级检查
**位置：** `frameless_window.py:274-291`

```python
def mousePressEvent(self, event):
    if event.button() == Qt.MouseButton.LeftButton:
        local_pos = event.position()
        if local_pos is not None:
            # 优先检查是否在顶部边缘
            if self._is_on_top_edge(local_pos.toPoint()) and self._window:
                # 在顶部边缘：忽略事件，让父窗口处理调整大小
                event.ignore()
                return

        # 不在边缘：开始拖动
        self._state_mgr.pressed = True
        pos = event.globalPosition()
        if pos is not None:
            self._drag_position = pos.toPoint()
```

**效果：** 顶部边缘（6px 内）触发调整大小而非拖动

## 设计原则

### 状态管理原则

**错误做法：**
```python
# 所有状态统一用 StateManager 管理
self._state_mgr.pressed  # 用于拖动和调整
```

**正确做法：**
```python
# 不同功能的状态独立管理
self._resizing           # 边缘调整大小（FramelessWindow）
title_bar._state_mgr.pressed  # 标题栏拖动（TitleBar）
```

### 事件传播原则

**边缘调整 > 标题栏拖动**

当两者冲突时（如顶部边缘），边缘调整优先：
1. 子组件（TitleBar）检测边缘
2. 如果在边缘：`event.ignore()` 让父组件处理
3. 如果不在边缘：子组件处理事件
