## 架构改进

### 职责划分

| 包 | 职责 | 抽象层次 | 示例 |
|---|------|---------|------|
| **core/** | 核心基础设施、工具类 | 底层抽象 | ThemeManager, AnimationController |
| **components/** | 原子UI组件 | 中层抽象 | CustomButton, ModernLineEdit |
| **containers/** | 容器组件 | 高层组件 | FramelessWindow |

### 依赖关系

```
应用层 (examples/)
    ↓
容器层 (containers/) ← 依赖 core
    ↓
组件层 (components/) ← 依赖 core
    ↓
核心层 (core/) ← 无业务依赖
```

## 为什么这样重构？

### 问题分析

**之前的结构问题**：
```
src/
└── core/
    ├── theme_manager.py       # ✅ 核心基础设施
    ├── animation_controller.py # ✅ 核心基础设施
    └── frameless_window.py    # ❌ 高级UI组件 - 职责不匹配
```

**FramelessWindow 不是"核心基础设施"**：
- ✅ 它是一个**容器组件**
- ✅ 它**依赖** core 的基础设施
- ✅ 它是**高层抽象**，不是底层工具
- ✅ 它是**业务组件**，不是通用工具

### 重构优势

| 优势 | 说明 |
|-----|------|
| **职责清晰** | core 专注基础设施，containers 专注容器组件 |
| **易于理解** | 新开发者能快速理解项目结构 |
| **便于扩展** | 添加新容器有明确位置 |
| **符合设计原则** | 遵循单一职责和依赖倒置原则 |

### 重构内容

#### 1. 样式解耦（Style Decoupling）

**问题：** 原始代码中 QSS 样式字符串硬编码在组件内部

**解决方案：** 提取独立的样式生成器类

```python
# 新增：src/core/frameless_window_styles.py
class TitleBarStyleGenerator:
    """TitleBar 样式生成器"""

    @staticmethod
    def generate_complete_qss(theme: Theme, is_maximized: bool) -> str:
        """从主题数据生成完整 QSS 字符串"""
        ...

class WindowStyleGenerator:
    """FramelessWindow 样式生成器"""

    @staticmethod
    def generate_complete_qss(theme: Theme, is_maximized: bool) -> str:
        """从主题数据生成完整 QSS 字符串"""
        ...
```

**改进效果：**
- ✓ 样式逻辑完全独立
- ✓ 可单独测试样式生成
- ✓ 支持运行时样式切换
- ✓ 符合单一职责原则

#### 2. 状态管理统一（State Management）

**问题：** 状态分散在多个布尔变量中
```python
# 旧代码
self._dragging = False
self._resizing = False
self._is_maximized = False
```

**解决方案：** 引入 StateManager 统一管理
```python
# 新代码
from core.state_manager import StateManager

class TitleBar(QWidget):
    def __init__(self):
        self._state_mgr = StateManager(self)
        # 使用 self._state_mgr.pressed 代替 self._dragging

class FramelessWindow(QWidget):
    def __init__(self):
        self._state_mgr = StateManager(self)
        # 使用 self._state_mgr.pressed 代替 self._resizing
```

**改进效果：**
- ✓ 统一的状态接口
- ✓ 自动信号通知机制
- ✓ 支持状态组合（hover + pressed）
- ✓ 符合项目的统一架构

#### 3. 动画控制器集成（Animation Controller）

**问题：** 无动画支持

**解决方案：** 预留 AnimationController 集成点

```python
from core.animation_controller import AnimationController

class TitleBar(QWidget):
    def __init__(self):
        self._animator = AnimationController(self, self)

    def _setup_animations(self):
        """预留动画配置方法"""
        # 未来可添加按钮缩放、颜色过渡等动画
        pass
```

**改进效果：**
- ✓ 建立了动画扩展点
- ✓ 为后续增强做好准备
- ✓ 符合开闭原则（对扩展开放）

#### 4. 代码结构改进

##### 信号驱动架构
```python
# TitleBar 现在通过信号通信
class TitleBar(QWidget):
    minimizeRequested = pyqtSignal()
    maximizeRequested = pyqtSignal()
    closeRequested = pyqtSignal()

# FramelessWindow 连接信号
self.title_bar.minimizeRequested.connect(self.showMinimized)
self.title_bar.maximizeRequested.connect(self._toggle_maximize)
self.title_bar.closeRequested.connect(self.close)
```

##### Q_PROPERTY 支持
```python
class FramelessWindow(QWidget):
    @pyqtProperty(bool)
    def isMaximizedState(self) -> bool:
        """导出最大化状态供动画使用"""
        return self._is_maximized
```

## 设计模式应用

| 模式 | 应用位置 | 代码行 |
|------|---------|--------|
| **单一职责原则** | StyleGenerator 类只负责样式生成 | `frameless_window_styles.py:18-75` |
| **开闭原则** | 预留动画扩展点 | `frameless_window.py:223-227` |
| **依赖倒置** | 依赖 Theme 抽象而非具体实现 | `frameless_window_styles.py:35` |
| **单一职责原则** | 状态管理委托给 StateManager | `frameless_window.py:147` |
| **观察者模式** | 主题变更订阅/通知 | `frameless_window.py:164` |
| **信号驱动** | TitleBar 与 Window 松耦合通信 | `frameless_window.py:210-213` |
