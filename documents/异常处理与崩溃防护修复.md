# 主题切换崩溃问题 - 完整修复方案

## 问题描述
点击demo中的主题切换按钮会导致程序崩溃。

## 修复内容

### 1. 添加 `set_theme()` 便捷方法
**文件**: `src/core/theme_manager.py`

**问题**: ThemeManager只有 `set_current_theme()` 方法，但所有demo都调用 `set_theme()`

**修复**: 添加便捷方法作为别名
```python
def set_theme(self, theme_name: str) -> None:
    """Set active theme (convenience method)."""
    self.set_current_theme(theme_name)
```

### 2. 增强 ThemeManager 异常处理
**文件**: `src/core/theme_manager.py`

**改进**:
- 更详细的错误日志
- 检查widget是否有效
- 完整的堆栈跟踪
- 自动清理无效订阅者

```python
def _notify_subscribers(self) -> None:
    to_remove = []
    for widget, callback in self._subscribers.items():
        try:
            # 检查widget是否仍然有效
            if widget is None or not hasattr(widget, '__class__'):
                to_remove.append(widget)
                continue

            callback(self._current_theme)

        except RuntimeError as e:
            logger.error(f"Subscriber deleted: {e}")
            traceback.print_exc()
            to_remove.append(widget)
        except Exception as e:
            logger.error(f"Subscriber error: {e}")
            traceback.print_exc()
            to_remove.append(widget)

    # 清理无效订阅者
    for widget in to_remove:
        self.unsubscribe(widget)
```

```python
    except Exception as e:

        import traceback
        traceback.print_exc()
```

## 崩溃原因分析

### 可能的崩溃原因:

1. **访问已删除的widget**
   - Widget被删除后仍在订阅者列表中
   - 主题切换时尝试通知已删除的widget
   - **解决**: 检查widget有效性，添加异常处理

2. **样式表应用错误**
   - 某些组件在特定状态下应用样式表失败
   - **解决**: 捕获异常并记录，不影响其他组件

3. **属性访问错误**
   - 组件在某些状态下缺少必要属性
   - **解决**: try-except保护每个组件的_on_theme_changed

## 防护措施

现在的多层防护机制：

```
1. ThemeManager._notify_subscribers()
   ├─ 检查widget有效性
   ├─ 捕获RuntimeError (widget已删除)
   ├─ 捕获Exception (其他错误)
   └─ 自动清理无效订阅者

2. Component._on_theme_changed()
   ├─ 捕获所有异常
   ├─ 记录详细错误日志
   └─ 不影响其他组件
```
