# Top Edge Shrink Bug Fix

## 问题描述

**Bug现象：** 当窗口已经缩小到最小尺寸（400x300）时，继续拖动顶部边缘会使窗口向下移动，而不是保持固定位置。

**根本原因：** 在`_resize_window`方法中，虽然有防止窗口缩小到最小尺寸以下的逻辑，但没有完全阻止位置变化。当窗口达到最小尺寸后，继续拖动仍会触发几何位置的微小变化。

## 修复方案

### 修改位置
文件：`src/core/frameless_window.py`
方法：`_resize_window()`

### 关键改动

1. **强化边界检查逻辑**
```python
# 原逻辑 - 不够严格
if 'top' in self._edge and delta.y() > 0:
    return  # 只检查单一条件

# 修复后 - 更严格的检查
if ('top' in self._edge and delta.y() > 0) or \
   ('bottom' in self._edge and delta.y() < 0):
    return  # 同时检查多个条件
```

2. **添加明确的注释说明**
```python
# CRITICAL: Check if already at minimum size and trying to shrink further
# This prevents window movement when at minimum size
```

3. **优化条件判断结构**
- 使用更清晰的逻辑组合
- 添加详细的注释说明每个检查的目的
- 确保所有边缘方向都被正确处理

## 技术细节

### 最小尺寸约束
- 宽度最小值：400像素
- 高度最小值：300像素
- 边缘检测范围：6像素

### 逻辑流程
1. 检查当前窗口尺寸是否已达最小值
2. 判断拖动方向是否会进一步缩小窗口
3. 如果是，则完全阻止几何变化
4. 否则，正常执行尺寸调整

### 关键代码片段
```python
# 水平边缘检查（左/右）
if 'left' in self._edge or 'right' in self._edge:
    if current_w <= min_w:
        if ('left' in self._edge and delta.x() > 0) or \
           ('right' in self._edge and delta.x() < 0):
            return  # 阻止进一步缩小

# 垂直边缘检查（上/下）
if 'top' in self._edge or 'bottom' in self._edge:
    if current_h <= min_h:
        if ('top' in self._edge and delta.y() > 0) or \
           ('bottom' in self._edge and delta.y() < 0):
            return  # 阻止进一步缩小
```
