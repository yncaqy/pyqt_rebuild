# 无边框窗口顶部边缘调整大小修复

## 修复日期
2026-02-15

## 问题描述

### 症状
无边框窗口的**顶部边缘无法使用鼠标拖动调整大小**

当鼠标移动到窗口顶部边缘时：
- ❌ 无法触发调整大小功能
- ❌ 光标不改变
- ❌ 点击后会拖动窗口而不是调整大小

### 根本原因

`TitleBar` 类在重构时**遗漏了关键的边缘检测逻辑**：

```python
# 旧的 TitleBar.mousePressEvent() - 有问题
def mousePressEvent(self, event):
    if event.button() == Qt.MouseButton.LeftButton:
        self._dragging = True  # 总是开始拖动！
        ...
```

**问题：** TitleBar 总是处理鼠标事件并开始拖动，即使在顶部边缘也是如此。

---

## 修复方案

### 事件优先级机制

```
用户点击 TitleBar 顶部边缘 (y ≤ 6px)
    ↓
TitleBar.mousePressEvent()
    ↓
检测: _is_on_top_edge() 返回 True
    ↓
关键: event.ignore() ← 忽略事件！
    ↓
事件传播到 FramelessWindow
    ↓
FramelessWindow.mousePressEvent()
    ↓
_get_resize_edge() 检测到 'top'
    ↓
_resizing = True
    ↓
开始调整大小 ✅
```

### 代码修改

**文件：** `src/core/frameless_window.py`

#### 修改 1：添加边缘边距变量

**位置：** TitleBar.__init__() (第 136 行)

```python
def __init__(self, parent=None):
    super().__init__(parent)
    self.setFixedHeight(40)
    self._theme_mgr = ThemeManager.instance()
    self._icon = None
    self._dragging = False
    self._drag_position = QPoint()
    self._window = None
    self._edge_margin = 6  # ← 新增：边缘检测边距

```

#### 修改 2：添加顶部边缘检测方法

**位置：** TitleBar 类 (第 288-299 行)

```python
def _is_on_top_edge(self, pos: QPoint) -> bool:
    """检测是否在顶部边缘（用于调整大小）

    Args:
        pos: 标题栏内的本地位置

    Returns:
        如果 y 坐标在边距内则返回 True
    """
    margin = self._edge_margin
    return pos.y() <= margin
```

#### 修改 3：修改鼠标按下事件

**位置：** TitleBar.mousePressEvent() (第 301-313 行)

```python
def mousePressEvent(self, event):
    """处理鼠标按下事件（拖动或调整大小）"""
    if event.button() == Qt.MouseButton.LeftButton:
        local_pos = event.position()
        if local_pos is not None:
            # 优先检查：如果在顶部边缘，忽略事件让父组件处理调整大小
            if self._is_on_top_edge(local_pos.toPoint()) and self._window:
                event.ignore()  # ← 关键：让事件传播到父组件
                return

        # 不在边缘：开始拖动
        self._dragging = True
        pos = event.globalPosition()
        self._drag_position = pos.toPoint() if pos is not None else QPoint()
```

## 技术细节

### 事件传播机制

在 Qt 中，事件处理遵循以下规则：

1. **子组件优先接收事件**
   - TitleBar 先于 FramelessWindow 接收鼠标事件

2. **事件处理的两种方式**
   - `event.accept()` - 事件被处理，停止传播
   - `event.ignore()` - 事件被忽略，继续传播给父组件

3. **我们的修复利用了这个机制**
   - 在顶部边缘：`event.ignore()` → 传播到 FramelessWindow 处理调整大小
   - 在其他区域：正常处理 → TitleBar 处理拖动

### 边缘检测区域

```python
_edge_margin = 6  # 像素
```

- 顶部 6px 区域：触发调整大小
- 标题栏其他区域：触发窗口拖动
- 这个边距与 FramelessWindow 的 `_edge_margin` 一致

### 优先级原则

**边缘调整 > 标题栏拖动**

当两者冲突时（如顶部边缘），边缘调整功能优先：
- 确保用户体验一致
- 所有边缘都可以调整大小
- 符合操作系统标准行为
