## 修复的问题

### 1. 方法名不匹配
- **问题**: ThemeManager只有 `set_current_theme()`，demo调用 `set_theme()`
- **修复**: 添加 `set_theme()` 作为便捷方法
- **文件**: `src/core/theme_manager.py`

### 2. 缺少异常处理
- **问题**: 组件主题切换时遇到错误会导致整个程序崩溃
- **修复**: 所有7个组件的 `_on_theme_changed()` 都添加了try-except
- **文件**:
  - `src/components/buttons/custom_push_button.py`
  - `src/components/checkboxes/custom_check_box.py`
  - `src/components/inputs/modern_line_edit.py`
  - `src/components/progress/circular_progress.py`
  - `src/components/sliders/animated_slider.py`
  - `src/components/toasts/toast.py`
  - `src/containers/frameless_window.py`

### 3. ThemeManager通知机制增强
- **问题**: 通知订阅者时没有充分检查widget有效性
- **修复**:
  - 检查widget是否为None
  - 检查widget是否有有效属性
  - 详细的错误日志和堆栈跟踪
  - 自动清理无效订阅者
- **文件**: `src/core/theme_manager.py`

## 防护机制

现在有三层防护防止主题切换崩溃：

### 第1层：ThemeManager检查
```python
if widget is None or not hasattr(widget, '__class__'):
    to_remove.append(widget)
    continue
```

### 第2层：ThemeManager异常捕获
```python
try:
    callback(self._current_theme)
except RuntimeError as e:  # Widget deleted
    to_remove.append(widget)
except Exception as e:     # Other errors
    to_remove.append(widget)
```

### 第3层：组件异常处理
```python
def _on_theme_changed(self, theme: Theme) -> None:
    try:
        self._apply_theme(theme)
    except Exception as e:
        logger.error(f"Error: {e}")
        traceback.print_exc()
```
