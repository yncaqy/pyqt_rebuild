# 无边框窗口最小尺寸拖动修复

## 修复日期
2026-02-15

## 问题描述

### 症状
当从顶部边缘向下拖动调整窗口大小到**最小高度（300px）后**，如果继续向下拖动：
- ❌ 会错误地触发窗口拖动
- ❌ 窗口会移动而不是保持固定

### 复现步骤
1. 将鼠标移到窗口顶部边缘（y ≤ 6px）
2. 点击并向下拖动，缩小窗口
3. 窗口缩小到最小高度（300px）后停止
4. **继续向下拖动鼠标**
5. ❌ 窗口开始移动（触发拖动）

### 根本原因

**状态管理不完善**：当窗口达到最小尺寸后继续拖动时，`TitleBar` 的拖动逻辑没有正确检查 `FramelessWindow` 的调整大小状态。

```python
# 旧的 TitleBar.mouseMoveEvent() - 有问题
def mouseMoveEvent(self, event):
    if self._dragging and event.buttons() == Qt.MouseButton.LeftButton:
        # 没有检查 window._resizing 状态！
        # 即使正在调整大小，也可能触发拖动
        self._window.move(new_pos)
```

**问题场景**：
```
1. 用户在顶部边缘按下 → _resizing = True, title_bar._dragging = False
2. 向下拖动到最小尺寸
3. 继续拖动 → _resize_window() 限制尺寸，窗口不改变
4. 鼠标位置改变 → 可能重新进入某些检测区域
5. 缺少保护 → title_bar._dragging 被设置为 True
6. 触发窗口移动 ❌
```

## 修复方案

### 添加三层保护机制

**原则：** 在调整大小时完全阻止标题栏拖动逻辑

#### 保护层 1：TitleBar.mousePressEvent()

**文件：** `src/core/frameless_window.py` (第 301-318 行)

```python
def mousePressEvent(self, event):
    """Handle mouse press for window dragging."""
    if event.button() == Qt.MouseButton.LeftButton:
        # CRITICAL: Don't start dragging if window is being resized
        if self._window and hasattr(self._window, '_resizing') and self._window._resizing:
            event.ignore()  # ← 新增：阻止在调整大小时开始拖动
            return

        local_pos = event.position()
        if local_pos is not None:
            # Priority check: if on top edge, ignore event so parent can handle resize
            if self._is_on_top_edge(local_pos.toPoint()) and self._window:
                event.ignore()
                return

        # Not on edge and not resizing: start dragging
        self._dragging = True
        pos = event.globalPosition()
        self._drag_position = pos.toPoint() if pos is not None else QPoint()
```

**保护作用：**
- 防止在调整大小时通过新的鼠标按下事件开始拖动
- 即使用户在调整大小时重新点击鼠标

#### 保护层 2：TitleBar.mouseMoveEvent()

**文件：** `src/core/frameless_window.py` (第 320-333 行)

```python
def mouseMoveEvent(self, event):
    """Handle mouse move for window dragging."""
    try:
        # CRITICAL: Don't drag if window is being resized
        if self._window and hasattr(self._window, '_resizing') and self._window._resizing:
            return  # ← 新增：阻止在调整大小时继续拖动

        if self._dragging and event.buttons() == Qt.MouseButton.LeftButton:
            if self._window and not self._window.isMaximized():
                pos = event.globalPosition()
                if pos is not None:
                    current_pos = pos.toPoint()
                    delta = current_pos - self._drag_position
                    new_pos = self._window.pos() + delta
                    self._window.move(new_pos)
                    self._drag_position = current_pos
    except Exception as e:
        print(f"TitleBar mouseMoveEvent error: {e}")
        self._dragging = False
```

**保护作用：**
- 防止在调整大小时通过鼠标移动事件触发拖动
- 即使 `_dragging` 状态被错误设置，也会阻止实际移动

#### 保护层 3：FramelessWindow.mouseReleaseEvent()

**文件：** `src/core/frameless_window.py` (第 605-624 行)

```python
def mouseReleaseEvent(self, event):
    """Handle mouse release."""
    if event.button() == Qt.MouseButton.LeftButton:
        self._resizing = False
        self._edge = 'none'

        # Release mouse grab
        self.releaseMouse()

        # CRITICAL: Ensure TitleBar dragging state is reset
        if hasattr(self, 'title_bar') and self.title_bar:
            self.title_bar._dragging = False  # ← 新增：确保完全重置状态

        # Reset cursor to current edge state after resize
        try:
            pos = event.position()
            if pos is not None:
                edge = self._get_resize_edge(pos.toPoint())
                self._update_cursor(edge)
        except Exception:
            pass
```

**保护作用：**
- 确保释放鼠标时完全重置所有状态
- 防止状态残留影响下次操作

## 技术细节

### 状态同步机制

**调整大小时的状态设置：**
```python
# FramelessWindow.mousePressEvent()
self._resizing = True
# 同时清除 TitleBar 拖动状态
self.title_bar._dragging = False
```

**拖动时的状态检查：**
```python
# TitleBar 检查 FramelessWindow 的状态
if self._window and self._window._resizing:
    # 不执行拖动逻辑
```

**释放鼠标时的状态重置：**
```python
# FramelessWindow.mouseReleaseEvent()
self._resizing = False
self._edge = 'none'
self.title_bar._dragging = False  # 确保完全重置
```

### 为什么需要三层保护？

**单一保护不足的原因：**

1. **只有 mousePressEvent 检查**
   - ❌ 无法阻止已经在拖动状态下的 mouseMoveEvent
   - ❌ 如果 `_dragging` 被错误设置，会继续触发移动

2. **只有 mouseMoveEvent 检查**
   - ❌ 无法阻止新的 mousePressEvent 重新设置 `_dragging = True`
   - ❌ 状态可能被重新激活

3. **只有 mouseReleaseEvent 重置**
   - ❌ 只在释放鼠标时清理，太晚了
   - ❌ 在拖动过程中状态可能已经混乱

**三层保护的协同作用：**
```
mousePressEvent (阻止启动)
      ↓
mouseMoveEvent (阻止执行)
      ↓
mouseReleaseEvent (完全重置)
```
