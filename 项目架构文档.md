# PyQT_rebuild 项目架构文档

## 项目概述

PyQT_rebuild 是一个现代化的 PyQt6 组件库，展示了清晰的架构设计原则和最佳实践。

**设计理念**：
- 职责分离 (Separation of Concerns)
- 依赖倒置 (Dependency Inversion)
- 单一职责 (Single Responsibility)
- 开闭原则 (Open/Closed Principle)

## 核心包职责

### 1. core/ - 核心基础设施

**职责**：提供底层基础设施和工具类，不涉及具体业务逻辑。

| 子包 | 职责 | 抽象层次 |
|-----|------|---------|
| **theme/** | 主题管理和应用 | 基础设施 |
| **animation/** | 动画控制和过渡 | 基础设施 |
| **state/** | 状态管理和同步 | 基础设施 |
| **painting/** | 绘制策略模式 | 设计模式 |
| **platform/** | 平台抽象层 | 跨平台抽象 |

**设计原则**：
- ✅ 被其他模块依赖，不依赖业务代码
- ✅ 提供稳定的 API 接口
- ✅ 支持依赖注入和单例模式
- ✅ 平台无关的抽象接口

**依赖规则**：
```
core → 不依赖任何业务代码
components → 依赖 core
containers → 依赖 core，可能依赖 components
```

---

### 2. components/ - 原子UI组件

**职责**：提供可复用的原子UI组件，每个组件独立且职责单一。

| 组件类别 | 功能 | 特点 |
|---------|------|------|
| **buttons/** | 按钮组件 | 主题化、动画 |
| **inputs/** | 输入组件 | 验证、样式 |
| **sliders/** | 滑块组件 | 动画过渡 |
| **checkboxes/** | 复选框 | 自定义样式 |
| **progress/** | 进度指示 | 可插拔绘制 |
| **toasts/** | 通知消息 | 非侵入式 |

**设计原则**：
- ✅ 每个组件独立可测试
- ✅ 支持主题系统集成
- ✅ 使用策略模式（如绘制策略）
- ✅ 最小化外部依赖

**组件模板**：
```python
class CustomComponent(QWidget):
    """
    Brief description.

    Features:
    - Feature 1
    - Feature 2
    """

    def __init__(self, parent=None):
        super().__init__(parent)
        # 初始化主题管理器
        self._theme_mgr = ThemeManager.instance()
        # 订阅主题变化
        self._theme_mgr.subscribe(self, self._on_theme_changed)

    def _on_theme_changed(self, theme):
        # 应用主题
        pass
```

---

### 3. containers/ - 容器组件

**职责**：提供高级容器组件，组合多个UI元素，提供复杂功能。

**当前组件**：
- **FramelessWindow**：无边框窗口容器
  - 自定义标题栏
  - 边缘调整大小
  - 平台特定功能（Windows 11 圆角等）
  - 完整的 QWidget 风格 API

**设计原则**：
- ✅ 组合多个原子组件
- ✅ 提供完整的用户体验
- ✅ 依赖 core 基础设施
- ✅ 可能依赖 components

**为什么 FramelessWindow 在 containers/ 而不是 core/**：

| 方面 | core 的职责 | FramelessWindow 的性质 |
|------|------------|---------------------|
| 职责 | 基础设施、工具类 | 高级UI组件、容器 |
| 抽象层次 | 底层抽象 | 高层组件 |
| 依赖关系 | 被其他模块依赖 | 依赖 core 的基础设施 |
| 使用场景 | 作为基础工具被使用 | 作为完整组件被使用 |

## 设计模式应用

### 1. 策略模式 (Strategy Pattern)

**应用场景**：组件绘制

```python
# core/painting/widget_painter.py
class WidgetPainter:
    """绘制策略基类"""
    def paint(self, widget, painter):
        raise NotImplementedError

# core/painting/circular_progress_painter.py
class CircularProgressPainter(WidgetPainter):
    """圆形进度条绘制策略"""
    def paint(self, widget, painter):
        # 具体绘制实现
        pass
```

**优势**：
- 可插拔的绘制策略
- 运行时切换绘制方式
- 易于扩展新的绘制风格

---

### 2. 单例模式 (Singleton Pattern)

**应用场景**：主题管理器

```python
# core/theme/theme_manager.py
class ThemeManager:
    _instance = None

    @classmethod
    def instance(cls):
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
```

**优势**：
- 全局唯一的主题状态
- 统一的主题切换入口
- 观察者模式的实现基础

---

### 3. 观察者模式 (Observer Pattern)

**应用场景**：主题变更通知

```python
# 订阅主题变化
ThemeManager.instance().subscribe(widget, widget._on_theme_changed)

# 发布主题变化
ThemeManager.instance().set_theme('dark')
```

**优势**：
- 解耦主题发布者和订阅者
- 自动同步主题状态
- 支持多个订阅者

---

### 4. 工厂模式 (Factory Pattern)

**应用场景**：平台抽象层

```python
# core/platform/__init__.py
def get_platform() -> WindowPlatform:
    """获取平台特定的实现"""
    platform_name = sys.platform

    if platform_name == 'win32':
        from .windows import WindowsPlatform
        return WindowsPlatform()
    elif platform_name == 'darwin':
        from .darwin import DarwinPlatform
        return DarwinPlatform()
    else:
        return BasePlatform()
```

**优势**：
- 平台无关的接口
- 运行时平台检测
- 优雅降级机制

## 依赖管理

### 依赖规则

1. **单向依赖**：上层可以依赖下层，下层不能依赖上层
   ```
   ✅ containers → core
   ✅ components → core
   ✅ containers → components
   ❌ core → containers
   ❌ core → components
   ```

2. **接口依赖**：依赖抽象接口，不依赖具体实现
   ```python
   # core/platform/__init__.py
   class WindowPlatform(Protocol):
       def set_corner_preference(self, hwnd, rounded): ...

   # 依赖接口，不依赖具体实现
   platform: WindowPlatform = get_platform()
   ```

3. **依赖注入**：通过构造函数或方法注入依赖
   ```python
   class CustomWidget(QWidget):
       def __init__(self, theme_manager: ThemeManager):
           self._theme_mgr = theme_manager
   ```

## 扩展指南

### 添加新的原子组件

1. 在 `components/` 下创建对应目录
2. 实现组件类，继承 `QWidget`
3. 集成主题系统
4. 编写示例和测试

**示例**：
```python
# components/cards/card.py
from core.theme_manager import ThemeManager
from PyQt6.QtWidgets import QWidget, QVBoxLayout, QLabel

class Card(QWidget):
    def __init__(self, parent=None):
        super().__init__(parent)
        self._theme_mgr = ThemeManager.instance()
        self._init_ui()

        # 订阅主题变化
        self._theme_mgr.subscribe(self, self._apply_theme)

    def _init_ui(self):
        layout = QVBoxLayout(self)
        self.title_label = QLabel("Card Title")
        layout.addWidget(self.title_label)

    def _apply_theme(self, theme):
        # 应用主题
        bg = theme.get_color('card.background', QColor(255, 255, 255))
        self.setStyleSheet(f"background-color: {bg.name()};")
```

### 添加新的容器组件

1. 在 `containers/` 下创建新文件
2. 组合现有的原子组件
3. 提供完整的用户体验

**示例**：
```python
# containers/splitter_container.py
class SplitterContainer(QWidget):
    """可分割的容器组件"""
    def __init__(self, parent=None):
        super().__init__(parent)
        from core.theme_manager import ThemeManager
        self._theme_mgr = ThemeManager.instance()
        # ... 实现细节
```

### 添加新的平台支持

1. 在 `core/platform/` 下创建平台文件
2. 实现 `WindowPlatform` 接口
3. 在 `__init__.py` 中注册

**示例**：
```python
# core/platform/linux.py
class LinuxPlatform:
    def set_corner_preference(self, hwnd, rounded):
        # Linux 特定实现
        pass

    def maximize_window(self, hwnd):
        # Linux 特定实现
        pass

# core/platform/__init__.py
def get_platform():
    if sys.platform == 'linux':
        from .linux import LinuxPlatform
        return LinuxPlatform()
    # ... 其他平台
```

## 最佳实践

### 1. 主题集成

所有组件都应该集成主题系统：

```python
from core.theme_manager import ThemeManager

class MyWidget(QWidget):
    def __init__(self, parent=None):
        super().__init__(parent)
        self._theme_mgr = ThemeManager.instance()
        self._theme_mgr.subscribe(self, self._apply_theme)
        self._apply_theme(self._theme_mgr.current_theme())

    def _apply_theme(self, theme):
        # 应用主题
        pass

    def cleanup(self):
        # 清理时取消订阅
        self._theme_mgr.unsubscribe(self)
```

### 2. 类型提示

所有公共方法都应该添加类型提示：

```python
from typing import Optional

def set_content(self, widget: Optional[QWidget]) -> None:
    """设置内容组件。

    Args:
        widget: 要设置的组件，None 表示清除
    """
    pass
```

### 3. 文档字符串

使用 Google 风格的文档字符串：

```python
def complex_method(self, param1: str, param2: int) -> bool:
    """简短描述。

    详细描述可以多行。

    Args:
        param1: 参数1的描述
        param2: 参数2的描述

    Returns:
        返回值的描述

    Raises:
        ValueError: 什么情况下抛出
    """
    pass
```

### 4. 日志使用

使用 logging 而不是 print：

```python
import logging

logger = logging.getLogger(__name__)

# Debug 级别 - 开发调试
logger.debug(f"Variable value: {value}")

# Info 级别 - 重要信息
logger.info("Component initialized")

# Warning 级别 - 警告但不影响运行
logger.warning("Failed to load optional feature")

# Error 级别 - 错误但可恢复
logger.error(f"Failed to save data: {e}")
```

## 性能优化原则

1. **缓存优先**：缓存计算结果，避免重复计算
2. **延迟加载**：按需加载资源
3. **事件节流**：限制高频事件的处理频率
4. **批量更新**：合并多次更新为一次

**示例**：
```python
# 边缘检测缓存
self._edge_cache = {}
cache_key = (pos.x(), pos.y())
if cache_key in self._edge_cache:
    return self._edge_cache[cache_key]
# 计算并缓存
result = self._calculate_edge(pos)
self._edge_cache[cache_key] = result
```
